Option 1: Railway (Recommended)

Railway is the best free option because:
  • 500 hours/month free (enough for 24/7 hosting)
  • Built-in PostgreSQL database
  • Custom domains included
  • Easy GitHub integration
  • Great performance and reliability

Step 1: Prepare Your Application

# Navigate to your project cd teztin-website # Install dependencies npm install # Test locally npm start

Step 2: Deploy to Railway

  1. Go to Railway.app and sign up with GitHub
  2. Create a new project
  3. Connect your GitHub repository
  4. Add PostgreSQL service (free tier)
  5. Set environment variables:
    NODE_ENV=production DB_HOST=your-railway-db-host DB_PORT=5432 DB_NAME=railway DB_USER=postgres DB_PASSWORD=your-railway-db-password
  6. Deploy automatically (Railway will detect your Node.js app)

Step 3: Access Your Application

  • Main Website: https://your-app.railway.app
  • Dashboard: https://your-app.railway.app/dashboard
  • API: https://your-app.railway.app/api/contacts

Option 2: Render (Alternative)

Step 1: Prepare Application

# Same preparation as Railway cd teztin-website npm install

Step 2: Deploy to Render

  1. Go to Render.com and sign up
  2. Create a new Web Service
  3. Connect your GitHub repository
  4. Add PostgreSQL database (free tier)
  5. Configure build settings:
    Build Command: npm install Start Command: npm start
  6. Set environment variables (same as Railway)
  7. Deploy

Option 3: Supabase + Vercel

Step 1: Set Up Supabase Database

  1. Go to Supabase.com and create account
  2. Create new project
  3. Get connection details from Settings > Database

Step 2: Deploy Backend to Vercel

# Install Vercel CLI npm i -g vercel # Deploy vercel # Set environment variables in Vercel dashboard

Step 3: Environment Variables

NODE_ENV=production DB_HOST=your-supabase-host DB_PORT=5432 DB_NAME=postgres DB_USER=postgres DB_PASSWORD=your-supabase-password

Environment Variables Setup

Required Variables (Free Platforms)

NODE_ENV=production DB_HOST=your-database-host DB_PORT=5432 DB_NAME=postgres (or your-db-name) DB_USER=postgres (or your-db-user) DB_PASSWORD=your-database-password

How to Get Database Credentials

  1. Railway: Go to your PostgreSQL service > Variables
  2. Supabase: Settings > Database > Connection string
  3. Neon: Dashboard > Connection details
  4. Render: Database > Environment variables

Troubleshooting Free Hosting

Common Issues:

1. App Goes to Sleep

  • Railway: Free tier apps sleep after inactivity
  • Solution: First request will wake up the app
  • Alternative: Use Render (longer sleep times)

2. Database Connection Issues

  • Check environment variables
  • Verify database is running
  • Test connection locally first

3. Build Failures

  • Check package.json has correct scripts
  • Verify all dependencies are listed
  • Check build logs for errors
