GitHub Setup & Free Deployment

Complete guide to get your local project on GitHub and deploy for free

Overview

This guide will help you:

  • Set up a GitHub account
  • Upload your local project to GitHub
  • Deploy to free hosting platforms
  • Get your website live for $0

Step 1: Create GitHub Account

1Go to GitHub.com

Visit https://github.com and click "Sign up"

2Create Account

Enter your email, create a password, and choose a username

3Verify Email

Check your email and click the verification link

Note: GitHub accounts are completely free. You only pay if you want private repositories or advanced features.

Step 2: Create GitHub Repository

1Click "New Repository"

On GitHub, click the "+" icon in the top right, then "New repository"

2Fill Repository Details

  • Repository name: teztin-website
  • Description: "Teztin contact system with enhanced dashboard"
  • Visibility: Public (free) or Private (free for students)
  • Don't check: "Add a README file" (we already have one)

3Click "Create Repository"

You'll see a page with setup instructions. Keep this open!

Step 3: Upload Your Project to GitHub

Your project is already initialized with Git!
We've already run git init and git commit for you.

1Add GitHub as Remote

Copy the repository URL from GitHub (it looks like: https://github.com/yourusername/teztin-website.git)

# Replace YOUR_USERNAME with your actual GitHub username git remote add origin https://github.com/YOUR_USERNAME/teztin-website.git

2Push to GitHub

git branch -M main git push -u origin main

3Verify Upload

Go to your GitHub repository page and you should see all your files!

Step 4: Deploy for Free

Now that your project is on GitHub, you can deploy it for free!

Option 1: Railway (Recommended)

1Go to Railway.app

Visit https://railway.app and sign up with GitHub

2Create New Project

Click "New Project" and select "Deploy from GitHub repo"

3Select Your Repository

Choose your teztin-website repository

4Add Database

Click "New" → "Database" → "PostgreSQL" (free tier)

5Set Environment Variables

Go to your web service → Variables and add:

NODE_ENV=production PG_URL=your-railway-postgresql-url

Get these values from: Your PostgreSQL service → Variables

6Deploy!

Railway will automatically deploy your app. Check the "Deployments" tab for progress.

Option 2: Render

1Go to Render.com

Visit https://render.com and sign up with GitHub

2Create Web Service

Click "New" → "Web Service" and connect your GitHub repository

3Configure Settings

  • Name: teztin-website
  • Build Command: npm install
  • Start Command: npm start

4Add Database

Click "New" → "PostgreSQL" (free tier)

5Set Environment Variables

Add the same environment variables as Railway

Step 5: Access Your Live Website

Congratulations! Your website is now live for free!

Your URLs:

  • Main Website: https://your-app.railway.app
  • Dashboard: https://your-app.railway.app/dashboard
  • API: https://your-app.railway.app/api/contacts

Test Your Deployment:

  1. Visit your main website URL
  2. Fill out the contact form
  3. Check the dashboard to see submissions
  4. Test the API endpoints

Troubleshooting

Common Issues:

1. "Repository not found"

  • Check your GitHub username is correct
  • Make sure the repository exists on GitHub
  • Verify you have access to the repository

2. "Authentication failed"

  • Use GitHub CLI: gh auth login
  • Or create a Personal Access Token
  • Or use SSH keys

3. "Build failed" on hosting platform

  • Check your package.json has correct scripts
  • Verify all dependencies are listed
  • Check the build logs for specific errors

4. "Database connection failed"

  • Check environment variables are set correctly
  • Verify database is running
  • Test connection locally first
