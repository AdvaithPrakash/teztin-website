How to Store "I'm Interested" Form Data

This guide walks you through setting up the database to store contact form submissions from your Teztin website.

Current Status

Right now: Form data is only displayed temporarily and logged to browser console. It's NOT being stored permanently.

Step-by-Step Setup

Step 1: Install PostgreSQL

First, you need PostgreSQL installed on your computer.

macOS:

brew install postgresql
brew services start postgresql

Windows:

Download from: https://www.postgresql.org/download/windows/

Linux:

sudo apt update
sudo apt install postgresql postgresql-contrib
sudo systemctl start postgresql

Step 2: Create Database

Create a database to store your contact form data.

# Connect to PostgreSQL
psql -U postgres

# Create the database
CREATE DATABASE teztin_contacts;

# Exit PostgreSQL
\q

Note: If you get a password prompt, try pressing Enter (empty password) or use 'postgres' as the password.

Step 3: Navigate to Your Project

cd teztin-website

Step 4: Install Dependencies

npm install

This installs all the required packages (Express, PostgreSQL driver, etc.)

Step 5: Set Up Environment Variables

# Copy the environment template
cp glassmorphism.env .env

# Edit the .env file with your database password
# Open .env in a text editor and update DB_PASSWORD

Important: Replace 'your_password' with your actual PostgreSQL password. If you haven't set a password, leave it empty.

Step 6: Start the Backend Server

node glassmorphism-backend.js

Step 7: Test the Form

  1. Open glassmorphism.html in your browser
  2. Fill out the contact form
  3. Click "Send Message"
  4. Check the success message - it should say "Data Saved to Database"

How to Access Your Stored Data

Method 1: API Endpoint (Easiest)

Visit this URL in your browser to see all form submissions:

http://localhost:3001/api/contacts

Method 2: Command Line

# Get all contacts via API
curl http://localhost:3001/api/contacts

# Or connect directly to database
psql -U postgres -d teztin_contacts

# View all contacts
SELECT * FROM contacts ORDER BY created_at DESC;

# Exit database
\q

Method 3: Database GUI Tool

Use a visual database tool like:

  • pgAdmin (Free, official PostgreSQL tool)
  • DBeaver (Free, supports multiple databases)
  • TablePlus (Paid, beautiful interface)

What Data Gets Stored

When someone submits the "I'm Interested" form, this information is saved:

  • Name (required)
  • Email (required)
  • Company (optional)
  • Project Description (required)
  • Submission Time (automatically added)
  • Unique ID (automatically generated)

Troubleshooting

Backend Won't Start

# Check if PostgreSQL is running
brew services list | grep postgresql

# Restart PostgreSQL
brew services restart postgresql

# Test database connection
psql -U postgres -d teztin_contacts -c "SELECT 1;"

Database Connection Error

# Check your .env file
cat .env

# Make sure DB_PASSWORD matches your PostgreSQL password
# Try empty password or 'postgres' if you haven't set one

Form Still Shows "Backend Not Available"

  • Make sure the backend is running (you should see the startup message)
  • Check that it's running on port 3001
  • Try refreshing the webpage
  • Check browser console (F12) for error messages

Sample Database Queries

View All Contacts

SELECT * FROM contacts ORDER BY created_at DESC;

View Recent Submissions (Last 24 Hours)

SELECT * FROM contacts 
WHERE created_at > NOW() - INTERVAL '24 hours';

Count Total Submissions

SELECT COUNT(*) FROM contacts;

View Contacts with Company Names

SELECT * FROM contacts 
WHERE company IS NOT NULL;
