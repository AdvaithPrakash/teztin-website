Teztin Database Setup Guide

How to Access Your Form Data

Current Status:

  • Form data is NOT currently stored in a database
  • Data is only logged to browser console and displayed temporarily
  • To store data permanently, you need to set up the backend

Option 1: Set Up Real Database (Recommended)

Step 1: Install PostgreSQL

# macOS (using Homebrew)
brew install postgresql
brew services start postgresql

# Windows
# Download from: https://www.postgresql.org/download/windows/

# Linux (Ubuntu/Debian)
sudo apt update
sudo apt install postgresql postgresql-contrib
sudo systemctl start postgresql

Step 2: Create Database

# Connect to PostgreSQL
psql -U postgres

# Create database
CREATE DATABASE teztin_contacts;

# Exit PostgreSQL
\q

Step 3: Set Up Backend

# Install dependencies
npm install express pg cors dotenv

# Copy environment file
cp glassmorphism.env .env

# Edit .env with your database credentials
# Update DB_PASSWORD with your actual PostgreSQL password

Step 4: Start Backend Server

# Start the backend
node glassmorphism-backend.js

# You should see: ðŸš€ Teztin backend running on port 3001
# And: âœ… Contacts table ready

Step 5: Test the Setup

  1. Open glassmorphism.html in your browser
  2. Fill out and submit the contact form
  3. Check the success message - it should show "Data Saved to Database"
  4. View all contacts: http://localhost:3001/api/contacts

How to Access Your Data

Method 1: API Endpoint

# Get all contacts
curl http://localhost:3001/api/contacts

# Or visit in browser:
http://localhost:3001/api/contacts

Method 2: Direct Database Access

# Connect to PostgreSQL
psql -U postgres -d teztin_contacts

# View all contacts
SELECT * FROM contacts ORDER BY created_at DESC;

# View recent contacts
SELECT name, email, company, created_at 
FROM contacts 
WHERE created_at > NOW() - INTERVAL '7 days';

# Exit
\q

Method 3: Database GUI Tools

  • pgAdmin (Free, official PostgreSQL tool)
  • DBeaver (Free, supports multiple databases)
  • TablePlus (Paid, beautiful interface)

Database Schema

CREATE TABLE contacts (
    id SERIAL PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    email VARCHAR(255) NOT NULL,
    company VARCHAR(255),
    description TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Sample Queries:

-- Get all contacts
SELECT * FROM contacts ORDER BY created_at DESC;

-- Get contacts from last 24 hours
SELECT * FROM contacts WHERE created_at > NOW() - INTERVAL '24 hours';

-- Count total contacts
SELECT COUNT(*) FROM contacts;

-- Get contacts by company
SELECT * FROM contacts WHERE company IS NOT NULL;

Troubleshooting

Backend Won't Start:

# Check if PostgreSQL is running
brew services list | grep postgresql

# Restart PostgreSQL
brew services restart postgresql

# Check connection
psql -U postgres -d teztin_contacts -c "SELECT 1;"

Database Connection Error:

# Check your .env file
cat .env

# Make sure password matches your PostgreSQL password
# Default password might be empty or 'postgres'

CORS Error:

  • Make sure the backend is running on port 3001
  • Check that CORS is properly configured in the backend

Alternative: No Database Setup

If you don't want to set up a database, the form will still work and show you the data locally:

  2. Page Display - Data shows on the page for 8 seconds

  1. Install PostgreSQL (if not already installed)
  2. Create database: createdb teztin_contacts
  3. Install dependencies: npm install

Need Help?

  • Database issues: Check PostgreSQL installation and credentials
  • Backend issues: Make sure all dependencies are installed
  • Form issues: Check browser console for error messages
  • CORS issues: Ensure backend is running on correct port

Your form data will be permanently stored and accessible via the API endpoint!
