Overview

This comprehensive guide covers deploying your Teztin contact form system with backend and database to various hosting platforms. The system includes:

  • Express.js backend with PostgreSQL database
  • Contact form with data collection
  • Enhanced dashboard with analytics
  • API endpoints for data export
  • Search and filtering capabilities

Prerequisites

Before deploying, ensure you have:
  • Node.js 18+ installed
  • Git installed and configured
  • Database (PostgreSQL) access
  • Hosting platform account
  • Basic knowledge of command line

Option 1: Heroku (Recommended for beginners)

Step 1: Prepare Your Application

# Navigate to your project directory cd teztin-website # Install dependencies npm install # Test locally npm start

Step 2: Set Up Database

  1. Create a PostgreSQL database:
    • Use Heroku Postgres add-on
    • Or use external service like Supabase, Railway, or Neon
  2. Get database credentials:
    • Host, port, database name, username, password
    • Connection string (if provided)

Step 3: Deploy to Heroku

# Install Heroku CLI # Download from: https://devcenter.heroku.com/articles/heroku-cli # Login to Heroku heroku login # Create Heroku app heroku create your-teztin-app # Add PostgreSQL add-on (if using Heroku Postgres) heroku addons:create heroku-postgresql:mini # Set environment variables heroku config:set NODE_ENV=production heroku config:set DB_HOST=your-db-host heroku config:set DB_PORT=5432 heroku config:set DB_NAME=your-db-name heroku config:set DB_USER=your-db-user heroku config:set DB_PASSWORD=your-db-password # Deploy git add . git commit -m "Deploy to Heroku" git push heroku main # Open your app heroku open

Step 4: Access Your Application

  • Main Website: https://your-teztin-app.herokuapp.com
  • Dashboard: https://your-teztin-app.herokuapp.com/dashboard
  • API: https://your-teztin-app.herokuapp.com/api/contacts

Option 2: Railway (Alternative to Heroku)

Step 1: Prepare Application

# Same as Heroku preparation cd teztin-website npm install

Step 2: Deploy to Railway

  1. Go to Railway.app
  2. Connect your GitHub repository
  3. Add PostgreSQL service
  4. Set environment variables:
    NODE_ENV=production DB_HOST=your-db-host DB_PORT=5432 DB_NAME=your-db-name DB_USER=your-db-user DB_PASSWORD=your-db-password

Step 3: Access Your Application

  • Main Website: https://your-app.railway.app
  • Dashboard: https://your-app.railway.app/dashboard

Option 3: Vercel (Frontend) + Supabase (Backend)

Step 1: Set Up Supabase Database

  1. Create Supabase account
  2. Create new project
  3. Get connection details

Step 2: Deploy Backend to Vercel

# Install Vercel CLI npm i -g vercel # Deploy vercel # Set environment variables in Vercel dashboard

Step 3: Deploy Frontend

Create vercel.json:

{ "version": 2, "builds": [ { "src": "glassmorphism-backend.js", "use": "@vercel/node" } ], "routes": [ { "src": "/(.*)", "dest": "/glassmorphism-backend.js" } ] }

Option 4: DigitalOcean App Platform

Step 1: Prepare Application

# Same preparation as other platforms cd teztin-website npm install

Step 2: Deploy

  1. Go to DigitalOcean App Platform
  2. Connect GitHub repository
  3. Add PostgreSQL database
  4. Set environment variables
  5. Deploy

Option 5: AWS/GCP/Azure (Advanced)

AWS Setup

# Install AWS CLI # Configure credentials # Deploy using Elastic Beanstalk eb init eb create eb deploy

GCP Setup

# Install Google Cloud CLI # Deploy to App Engine gcloud app deploy

Environment Variables

Required Variables

NODE_ENV=production DB_HOST=your-database-host DB_PORT=5432 DB_NAME=your-database-name DB_USER=your-database-user DB_PASSWORD=your-database-password PORT=3001 # Optional, will use platform's port

Optional Variables

CORS_ORIGIN=https://your-frontend-domain.com LOG_LEVEL=info

CREATE TABLE contacts ( id SERIAL PRIMARY KEY, name VARCHAR(255) NOT NULL, email VARCHAR(255) NOT NULL, company VARCHAR(255), description TEXT NOT NULL, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP );

Security Considerations

1. Environment Variables

  • Never commit .env files
  • Use platform-specific secret management
  • Rotate database passwords regularly

2. CORS Configuration

// Update CORS for production app.use(cors({ origin: process.env.CORS_ORIGIN || 'https://your-frontend-domain.com', credentials: true }));

3. Rate Limiting

npm install express-rate-limit
const rateLimit = require('express-rate-limit'); const limiter = rateLimit({ windowMs: 15 * 60 * 1000, // 15 minutes max: 100 // limit each IP to 100 requests per windowMs }); app.use(limiter);

Troubleshooting

Common Issues:

1. Database Connection Failed

  • Check environment variables
  • Verify database credentials
  • Test connection locally

2. CORS Errors

  • Update CORS origin in production
  • Check frontend domain

3. Port Issues

  • Use process.env.PORT in production
  • Don't hardcode port 3001

4. SSL Issues

  • Enable SSL in database connection
  • Use rejectUnauthorized: false for self-signed certificates
